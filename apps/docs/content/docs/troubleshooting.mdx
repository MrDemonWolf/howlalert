# Troubleshooting

## Mac not appearing on iPhone

1. Verify both devices are signed into the **same Apple ID**
2. Check that iCloud Drive is enabled on both devices
3. Restart HowlAlert on macOS (right-click menu bar icon → Quit, then relaunch)
4. On iPhone, pull-to-refresh the dashboard
5. Check System Settings → iCloud → ensure "iCloud Drive" is on

## Push notifications not arriving

1. Check iPhone Settings → Notifications → HowlAlert → ensure notifications are enabled
2. Verify your Pro subscription is active (Settings → Subscription in the app)
3. Ensure your Mac has internet access (pushes route through Cloudflare)
4. Wait 30 minutes — there's a cooldown per threshold to prevent spam

## Menu bar icon is grayscale

This means no active Pro subscription was found. Entitlement is checked via CloudKit.

1. Open HowlAlert on iPhone → Settings → verify subscription status
2. If recently subscribed, wait a few seconds for CloudKit sync
3. On Mac, click the wolf icon → check subscription status in the popover
4. If offline for 7+ days, the cached entitlement expires — reconnect to internet

## "Waiting for Mac..." on iPhone

The iPhone hasn't received any usage snapshots from a Mac yet.

1. Ensure HowlAlert is running on at least one Mac
2. Start a Claude Code session — HowlAlert needs JSONL activity to report
3. Check that `~/.claude/projects/` exists and contains `.jsonl` files

## Watch complication not updating

1. Ensure the HowlAlert watch app is installed (check Watch app on iPhone)
2. Verify the iPhone app is running in the background
3. Restart the watch app
4. Re-add the complication to your watch face

## High battery usage on Mac

HowlAlert uses FSEvents which is very efficient. If you see high CPU:

1. Check Activity Monitor for "HowlAlert" CPU usage
2. Restart the app
3. File a bug at [GitHub Issues](https://github.com/mrdemonwolf/howlalert/issues)
