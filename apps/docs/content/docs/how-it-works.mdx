# How It Works

## Data Flow

```
~/.claude/projects/**/*.jsonl
        │ FSEvents
        ▼
  macOS menu bar ──► Cloudflare Worker ──► APNs ──► iOS ──► Watch
       │                    │
       └─ CloudKit ◄────────┴─ D1 entitlement check
          (auto-pair)
```

## Step by Step

### 1. JSONL Watching (macOS)

Claude Code writes session data to `~/.claude/projects/` as JSONL files. HowlAlert watches these files using macOS FSEvents (the same API Finder uses).

From each JSONL entry, HowlAlert extracts:
- `output_tokens` — tokens generated by Claude
- `cache_read_input_tokens` — cached context tokens read
- `cache_creation_input_tokens` — new context tokens cached

Input tokens are **not** counted (they don't contribute to rate limits in the same way).

### 2. Pace Calculation

HowlAlert computes a "pace" — are you using tokens faster or slower than the window allows?

- **On track**: usage proportional to elapsed time
- **In debt**: burning faster than sustainable
- **In reserve**: plenty of headroom
- **Limit hit**: at or over 100%

### 3. CloudKit Sync

Each Mac writes its `UsageSnapshot` to your private iCloud database. This lets your iPhone aggregate usage across all your Macs.

No data leaves your iCloud. MrDemonWolf, Inc. cannot read your CloudKit records.

### 4. Push Notifications

When a threshold is crossed (60%, 85%, 100%), or when Claude finishes replying:

1. Your Mac sends a minimal request to our Cloudflare Worker
2. The Worker validates your subscription against D1
3. The Worker sends a push via Apple APNs
4. Your iPhone and Watch receive the notification

The Worker is **stateless** — it relays pushes and discards the request. Push cooldowns (30 min per threshold) prevent spam.

### 5. Entitlement

Your subscription is managed through RevenueCat on iOS. When you subscribe:
- RevenueCat webhook updates D1 (server-side truth)
- iOS writes an Entitlement record to CloudKit (client-side for instant Mac unlock)
- macOS reads CloudKit on launch + every 6 hours + on wake
